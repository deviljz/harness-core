# harness 0.3 全面优化-5 项

> 修复 v1.2 实战暴露的 5 个 harness-core 问题，bump 到 0.3.0

**complexity**: complex

---

## 1. Objective（目标）

**做什么**：harness-core 0.2.0 在喵辅导 v1.2 实战中暴露 5 个具体问题，本 spec 一次性修复并发版 0.3.0。

**5 个具体问题**：
1. **review-data JSON 编码 bug**：`harness review-data --spec X` 输出的 JSON 无法被 `python -m json.tool` 解析（`Expecting ',' delimiter`），diff_content 字段含未转义换行/反斜杠/引号
2. **/harness-full 自动连续跑机制不健全**：AI 在 subagent 中间输出 final text 时 Claude Code 主循环判停，无 hook 强制续跑
3. **Pass A review 模板默认依赖 diff，误报严重**：当 diff_content 不完整时（含 bug #1）大量"文件未在 diff"误报
4. **dart_test 在 harness check 显示 warn 而非 pass**：实际 flutter test 跑通了，但 reporter 解析不到 passed 数
5. **anti_patterns 无 ignore_paths**：全量扫描时扫到 scripts/ 和 tests/msqa/ 等历史代码目录，淹没真问题

**为什么做**：以上 5 项直接影响 /harness-full 体验。修完后下次跑 spec 更稳。

**用户是谁**：所有用 harness 的开发者 + AI agent

**成功标准**（可验证）：
1. `harness review-data --spec X | python -m json.tool` 解析成功
2. `/harness-full` 跑多 subagent 任务时，AI 中间输出文字也会被 Stop hook 拦截续跑（todo 未完成）
3. Pass A 模板加 Structure 文件清单核对（不只看 diff）
4. `flutter test` 跑通后 `harness check` 显示 dart_test status="pass"（不是 warn）
5. `anti_patterns` 段支持 `ignore_paths` 字段，全量扫描时跳过指定 glob
6. 单元测试全过；bump 0.2.0 → 0.3.0

**非目标**：
- 不重构 harness 核心（plan/execute/review 流程不动）
- 不引入新 LLM provider
- 不向后不兼容（老 config 必须仍能用）

## 6. Testing（测试策略）

每 fix 至少 1 个新 case：
- **fix #1**：跑 review-data 输出含 backslash / newline / quote 的 diff，json.loads 能解
- **fix #2**：active_tasks.json 文件存在且 todo 未完成 → hook exit 非 0 阻止 stop；文件不存在 → hook exit 0 允许 stop
- **fix #3**：Pass A 模板包含"Structure 文件清单核对"段
- **fix #4**：dart test stdout `"+5 -0" / "All tests passed!"` → reporter 解析为 passed=5
- **fix #5**：anti_patterns 段配 `ignore_paths: ["scripts/**"]`，全量扫描跳过 scripts/
